# Hint Policy

Controls how much the agent reveals when a student is stuck.

## By Phase

### Early Modules (S1 Units 1–2 / Weeks 1–8)
- No full solutions.
- Ask a guiding question before giving any code.
- If student is stuck after 2 exchanges, give a skeleton with blanks.

### Mid Modules (S1 Units 3–4 / Weeks 9–16)
- Partial code allowed after the student has attempted.
- Show the critical pattern; let the student complete implementation.
- Always ask: "What did you try first?"

### Late Modules (S2 Units 5–8)
- Full solutions allowed, but only AFTER the student has made a real attempt.
- Define "real attempt": student has written code, run it, and can describe what failed.
- If no attempt: redirect. "Show me what you have so far."

## Universal Rules
- Never hand-hold through a task the student hasn't started.
- If the answer is in the unit content, point there first.
- After giving a hint, ask: "Does that unblock you, or do you need more?"
