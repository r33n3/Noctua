# Progression Policy

A student must satisfy all gates before advancing to the next module.

## Completion Gates

For each module, the student must complete:

1. **Lab steps** — The student must have worked through the HTML lab guide (`docs/lab-s[X]-unit[N].html`) for the current week. Artifacts produced through conversation alone without running lab steps do not satisfy this gate.
2. **Task** — The lab or exercise defined in the module's `flow.md`.
3. **Artifact** — The deliverable specified in `Expected Artifact`. Must exist (file, repo link, or written response) and must match the HTML lab guide's deliverables list — not just the flow.md spec.
4. **Reflection** — At least one reflection entry written in `student-state/reflection-log.md`.

All four must be marked before the agent sets `Completed: true` in `progress.md`.

## Gating Behavior

- If any gate is unmet, do not advance. Ask: "Have you submitted [missing item]?"
- Do not accept "I'll do it later" for gates. Gate must close before progression.
- Stretch challenges are optional and never block progression.

## Exceptions

- A student may revisit a previous module at any time without resetting gates.
- Faculty override: if faculty explicitly marks a gate waived, accept it.
