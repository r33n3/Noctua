Side 1 · Foundation
Think → Spec → Build → Retro Cycle
THINK · 15 min
Problem definition. CCT + Assessment Stack. Write 3–5 measurable success criteria. Skill: /think → .claude/context.md
SPEC · 20 min
Architecture before code. /spec → SPEC.md (acceptance criteria, drift baseline). /build-spec → plans/SPEC-{name}.md (agents, tools, data flow, model selection, AIUC-1 controls). Cut scope now or pay later.
BUILD · 60 min
Structure (5) → Tool schemas (10) → Agents + prompts (15) → Orchestration (15) → Logging (10) → Test (5). Cut anything not in spec.
RETRO · 15 min
Happy path + failure case. Spec vs reality table. Top 3 items for next sprint. Skill: /retro → updates SPEC.md (drift) + CLAUDE.md (antipatterns)

Engineering Assessment Stack
LayerQuestionOptions
L1 · Problem What type? Classification · Correlation · Reasoning · Generation · Retrieval
L2 · Compute How solved? Deterministic (rules/regex) · Statistical (ML) · Reasoning (novel)
L3 · Model Which tier? Haiku · Sonnet · Opus — see model table below
L4 · Data Where stored? Relational (lookup) · Vector (semantic) · Graph (traverse) · Time-series (trends)
L5 · Integration How connected? Real-time MCP · Batch · A2A delegation · REST API
L6 · Verification How confirmed? Test suite (deterministic) · Human review (reasoning) · Cross-reference (factual)
Model Selection
ModelCostUse When
Haiku $1/MTok High-throughput routing, extraction, classification, triage
Sonnet $3/MTok Operational analysis, MCP tool orchestration, sprint builds
Opus $15/MTok Deep attribution, novel threat scenarios, adversarial reasoning. Invoke rarely.
Cost Optimization
StrategySavingsWhen to Use
Prompt Caching ~90% on repeated context Same system prompt, docs, or conversation history sent across multiple requests. Cache the prefix — only pay to process it once.
Batch Processing 50% on input + output High-volume workloads that don't need real-time responses. Send async, collect results later. Best for bulk analysis, eval runs, report generation.
Model Matching 3–15× per request Start dev on Sonnet. Evaluate Haiku for production volume tasks. Reserve Opus for genuine deep reasoning — not default escalation.
Rule: caching removes redundant processing cost · batching removes per-token cost · model matching removes per-request cost

CCT — 5 Critical Thinking Pillars
AccuracyIs the output factually correct? Verify before acting.
CompletenessIs anything important missing from the analysis?
ConsistencyDoes it contradict itself or prior outputs?
TraceabilityCan you follow the reasoning step by step?
CalibrationDoes expressed confidence match actual reliability?
V&V — 4 Dimensions
Output
Verification		 Check facts before acting. A tool producing 10 verified findings beats 50 unverified. VR tracks this.
Calibrated
Trust		 Match trust level to track record. High confidence on tested tasks, skepticism on novel ones.
Failure
Imagination		 Anticipate failure modes before they occur. What breaks under adversarial input?
Adversarial
Assumption		 Assume someone is trying to manipulate the system. Design for the attacker, not the happy path.

Side 3 · Field Guide
Protocols — What They Actually Do
MCP — Model Context Protocol
Agent calls a tool (function) you expose
Plain language: You write a function. Claude calls it. That's it. MCP is the standard wire format that lets any MCP-compatible agent call your function without custom glue code. Think of it as a shared plugin interface for AI.
A2A — Agent-to-Agent
One agent delegates work to another agent
Plain language: Instead of calling a tool, one agent hands a task to another specialized agent and waits for a result. Enables pipelines: triage agent → analysis agent → report agent. The handoff is structured JSON over HTTP, not a custom API per pair.
SPIFFE / SVIDs
Agent needs to prove its identity to a service
Plain language: Every agent gets a short-lived cryptographic ID card (SVID) issued by a trust authority. When the agent calls a service, it presents that card. No more shared API keys rotated by hand. If the agent is compromised, its card expires in minutes.
NHI — Non-Human Identity
Any service account, bot, or agent acting on behalf of a user
Plain language: NHI is the category of identity that isn't a person. Agents, pipelines, and CI runners are all NHIs. They need their own lifecycle: provisioned, scoped, rotated, revoked. Most orgs don't manage this well — it's a common attack surface.

CheckGood signRed flag

Harness Engineering — Plain Language

A harness makes correct agent behavior the easy path. When it's missing, agents drift. When it's present, agents self-correct.

HarnessConstrainsLives in
DevelopmentClaude while you work — session tool scope, file access, hook enforcement.claude/settings.json, .claude/hooks/
DeploymentThe agent you ship — input validation, output filtering, audit pipelineharnesses/<agent>/blueprint.yaml, fixed_steps/

Same vocabulary, different jobs. Don't conflate them.

LayerWhat it isWhat breaks without it
SpecWritten definition of what the agent must and must not do. Includes anti-requirements.Agent interprets intent differently every session.
RuntimeConstraints enforced at execution: tool scopes, output schemas, approval gates.Agent takes unintended high-impact actions.
ContextKnowledge the agent carries in: AGENTS.md, task state, prior retro lessons.Agent relearns the same lessons every session.
Three-Tier Context Architecture
TierPlain languageWhere
T1 · InstitutionalPermanent lessons about how this agent works in this codebase. Human-written. Surgical — only what can't be inferred from code.AGENTS.md
T2 · ProjectCurrent task state, database snapshots, handoff summaries. Changes per sprint. Acts as agent memory across sessions.Structured handoff files
T3 · SessionWhat fits in the context window right now. Constructed fresh each session from T1 + T2 + task prompt.Context window
Agent Memory Types
SensoryCurrent context window. Fast, bounded. Gone when session ends.
EpisodicRecords of past sessions: retro notes, what worked, what failed.
SemanticFactual knowledge base: domain patterns, codebase conventions, known traps.
ExternalDatabases, vector stores, knowledge graphs the agent can query at runtime.
Gap Classification (from /retro)
Spec gapThe spec didn't say it. Fix: add to spec + anti-requirements.
Constraint gapThe spec said it but runtime didn't enforce it. Fix: add tool scope or gate.
Context gapAgent didn't have the knowledge to succeed. Fix: add to AGENTS.md or T2 handoff.
Process gapWorkflow broke down (review skipped, retro missed). Fix: update checklist or gate.

Three Strikes Rule: Same gap type 3× across retros → build a permanent harness component.

Practitioner Assessment — Walking into a Company

Your goal: understand their current AI posture, identify gaps, and propose security agent deployments that add immediate value.

PhaseQuestions to ask
1 · Inventory What AI/LLM systems are deployed? Who owns each? Are agents listed in a registry?
2 · Identity How do agents authenticate? Are NHIs inventoried? Are credentials short-lived?
3 · Permissions What can each agent do? Is it least-privilege? Are tool scopes documented?
4 · Observability Are agent decisions logged? Can you reconstruct what an agent did and why?
5 · Harness Does each agent have a spec? AGENTS.md? Retro process? Or is it ad-hoc?
6 · Human gates What triggers human approval? Are irreversible actions gated?
7 · Threat model Has prompt injection been considered? Supply chain? Model theft? MITRE ATLAS run?
