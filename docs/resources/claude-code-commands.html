ESSENTIAL — Learn First
/compact
Compress conversation history. Add arg to specify what to keep: /compact keep the error patterns COST
Run at 80% context, not 95%. Auto-compact loses more detail.
/clear
Wipe conversation. File edits persist. Use between unrelated tasks.
/plan
Enter plan mode — read-only analysis, no file changes. Shift+Tab ×2 to toggle. ENFORCE
Claude can't edit files in plan mode. True enforcement.
/model
Switch model mid-session: opus, sonnet, haiku. Option 4 = Opus plan + Sonnet execute. COST
/cost
Show session token usage and cost breakdown. COST
Run after every lab exercise for cost awareness.
/help
List all available commands and skills. Start here when lost.
CONTEXT MANAGEMENT
/context
Show token usage breakdown — tools, MCP, skills, conversation. Color-coded grid. COST
/effort
Set reasoning depth: low ○ / medium ◐ / high ● / auto. NEW COST
Low for iteration, high for analysis. /effort auto resets.
/btw
Ask side question without touching context. Ephemeral, no tools, cache-only. NEW COST
Use mid-task: "/btw what file was that error in?"
/memory
View/edit persistent MEMORY.md. Claude auto-saves project conventions here.

Inline extended thinking. Also: "think hard", "ultrathink" in prompt text. COST

Reads: CLAUDE.md (CCT framework). Writes: conversation only.

Engineering Assessment Stack decision. Chat/Cowork/Code recommendation.

Reads: working tool. Writes: reports/AUDIT-{name}.md
/harness
Security Minion harness builder. Blueprint YAML + fixed/flexible steps.
Reads: plans/SPEC-*.md. Writes: harnesses/{type}/blueprint.yaml

Isolated git worktrees for parallel agents. Each gets own .claude/ copy.

Sprint retrospective. Lessons learned, what worked, what didn't.

KEYBOARD SHORTCUTS
Shift+Tab
Cycle: Normal → Auto-accept → Plan mode
Ctrl+C
Cancel current response. Claude stops mid-generation.
Ctrl+F ×2
Kill all background agents. Hit twice within 3 seconds to confirm.
Ctrl+G
Open plan in default editor. Edit before Claude executes.
Esc ×2
Open rewind menu. Choose conversation-only or code-only rollback.
↑
Browse prompt history. Current session first, then older sessions.
/vim
Enable vim keybindings in prompt input. Full modal editing.

DEFENSE LAYER CLASSIFICATION — Inside vs Outside the Reasoning Loop
⚠ INSIDE LOOP — GUIDANCE (probabilistic, bypassable)
CLAUDE.md — agent CAN ignore under goal pressure
AGENTS.md — guidance, not enforcement
.claude/rules/*.md — scoped, still probabilistic
.claude/skills/ — activation not guaranteed
.claude/agents/*.md — system prompts, may deviate
plans/*.md — intent docs, not constraints
harnesses/*/flexible_steps/ — the "AI part"
✓ OUTSIDE LOOP — ENFORCEMENT (deterministic, always fires)
settings.json → permissions.deny — CANNOT bypass
settings.json → hooks.PreToolUse — runs BEFORE tool
settings.json → hooks.PostToolUse — runs AFTER tool
settings.json → sandboxing — OS-level isolation
.claude/hooks/*.sh — the actual enforcement code
--max-turns, --max-budget-usd — hard caps
--allowedTools — explicit tool restriction
harnesses/*/fixed_steps/ — deterministic pipeline
harnesses/*/blueprint.yaml — defines the boundary
