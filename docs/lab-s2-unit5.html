def investigate(alert_text: str, title: str = "SOC Investigation") -> str:
    session = client.beta.sessions.create(
        agent=AGENT_ID,
        environment_id=ENV_ID,
        title=title,
    )
    print(f"Session: {session.id}\n")

    output = []
    # Open stream BEFORE sending — avoids race condition
    with client.beta.sessions.events.stream(session.id) as stream:
        client.beta.sessions.events.send(
            session.id,
            events=[{"type": "user.message", "content": [

            ]}],

            elif event.type == "session.status_idle":
                print("\n\n── Investigation complete ──")
                break

    return "".join(output)

A2A vs. MCP — Two Protocols, Two Jobs

Two protocols appear in agentic system architecture. They solve different problems:

  • MCP (Model Context Protocol) — how an agent calls a tool. Defines the interface between an agent and external capabilities: APIs, databases, file systems. You built an MCP server in S1 Unit 2.
  • A2A (Agent-to-Agent Protocol) — how agents communicate with each other. Defines message passing, task delegation, and result handoff between agents in a multi-agent system.

A production multi-agent system needs both: MCP to connect agents to tools, A2A to connect agents to each other. In Managed Agents, callable_agents[] handles A2A at one delegation level (coordinator → subagent). MCP is how each agent accesses tools — yours from Unit 2 included.

# runner_with_custom_tool.py — handle agent.custom_tool_use events
import anthropic, os, json, subprocess

client = anthropic.Anthropic()

MCP_SERVER_URL = "http://localhost:3000"

def call_mcp_server(tool_name: str, arguments: dict) -> str:
    """Call your local MCP server and return the result as a string."""
    import urllib.request
    payload = json.dumps({"tool": tool_name, "arguments": arguments}).encode()
    req = urllib.request.Request(
        f"{MCP_SERVER_URL}/call",
        data=payload,
        headers={"Content-Type": "application/json"},
    )
    with urllib.request.urlopen(req, timeout=10) as resp:
        return resp.read().decode()

def investigate_with_mcp(alert_text: str) -> str:
    session = client.beta.sessions.create(
        agent=os.environ["AGENT_V2_ID"],
        environment_id=os.environ["ENVIRONMENT_ID_PROD"],
        resources=[{
            "type": "memory_store",
            "memory_store_id": os.environ["MEMORY_STORE_ID"],
            "access": "read_write",
            "instructions": "Check /context/ for thresholds. Write findings to /findings/.",
        }],
    )

    events_by_id = {}
    report_parts = []

    with client.beta.sessions.events.stream(session.id) as stream:
        client.beta.sessions.events.send(
            session.id,
            events=[{"type": "user.message", "content": [
                {"type": "text", "text": alert_text}
            ]}],
        )
        for event in stream:
            events_by_id[event.id] = event

            if event.type == "agent.message":
                for block in event.content:
                    if hasattr(block, "text"):
                        print(block.text, end="", flush=True)
                        report_parts.append(block.text)
            elif event.type == "agent.custom_tool_use":
                print(f"\n[Custom tool: {event.name}({event.input})]", flush=True)
            elif event.type == "session.status_idle":
                stop = event.stop_reason
                if stop and stop.type == "requires_action":
                    # Agent called a custom tool — execute locally and return result
                    for event_id in stop.event_ids:
                        tool_event = events_by_id[event_id]
                        result = call_mcp_server(tool_event.name, tool_event.input)
                        client.beta.sessions.events.send(
                            session.id,
                            events=[{
                                "type": "user.custom_tool_result",
                                "custom_tool_use_id": event_id,
                                "content": [{"type": "text", "text": result}],
                            }],
                        )
                elif stop and stop.type == "end_turn":
                    print("\n── Investigation complete ──")
                    break

    return "".join(report_parts)

Platform vs. SDK — the correct mental model

Claude Managed Agents and Claude Agent SDK are two paradigms for running the same model:

  • Managed Agents (Weeks 1–2): Anthropic hosts the loop. Tools execute in a server-side container. You stream events. You don't see raw tool inputs/outputs.
  • Claude Agent SDK (this week): The loop runs in your Python process via query(). Tools execute locally. You see everything — every tool call, every result, every turn. Full observability.

OpenAI has the same split: Assistants API (hosted platform, equivalent to Managed Agents) vs. OpenAI Agents SDK (client-side library, equivalent to Claude Agent SDK). The Week 4 comparison isolates the infrastructure variable: same model, same task, hosted vs. client-side.

# sdk_basic.py
import asyncio
from claude_agent_sdk import query, ClaudeAgentOptions

SOC_SYSTEM_PROMPT = """You are a senior SOC analyst conducting incident investigations.
Given a security alert:
1. Extract and enrich all IoCs (IPs, domains, file hashes) using web search
2. Apply CCT analysis: map to MITRE ATT&CK, generate 3 hypotheses with probabilities
3. Assess severity (Critical/High/Medium/Low) with justification
4. Produce a structured JSON report followed by a concise narrative summary"""

async def investigate(alert_text: str) -> str:
    report_parts = []
    tool_calls = []

    async for message in query(
        prompt=alert_text,
        options=ClaudeAgentOptions(
            system_prompt=SOC_SYSTEM_PROMPT,
            allowed_tools=["WebSearch", "WebFetch", "Read", "Write"],
        ),
    ):
        # Every message, tool call, and result is visible here
        msg_type = type(message).__name__
        if hasattr(message, "content") and message.content:
            for block in message.content:
                if getattr(block, "type", None) == "text":
                    print(block.text, end="", flush=True)
                    report_parts.append(block.text)
                elif getattr(block, "type", None) == "tool_use":
                    # Full tool input visible client-side
                    tool_calls.append({"name": block.name, "input": block.input})
                    print(f"\n[Tool: {block.name}({block.input})]", flush=True)
        if hasattr(message, "result"):
            print(f"\n── Session complete ({len(tool_calls)} tool calls) ──")

    return "".join(report_parts)

with open("test_incidents.json") as f:
    import json
    incidents = json.load(f)

result = asyncio.run(investigate(incidents[0]["alert_text"]))

Deep Agents Exercise — Cross-Team Memory with Agent Diaries

WEEK 4Exercise: Persistent Cross-Agent Memory

Exercise Goal: Wire each MASS agent to maintain a persistent diary across scan runs. By the end of this exercise, your agent team accumulates institutional knowledge: patterns seen across multiple scans, recurring findings, false positive classifications, and inter-agent knowledge sharing. This closes the amnesia gap — the failure mode where each new scan starts from zero. (Theory: Week 4 — Institutional Memory for Agent Teams)

The amnesia problem. Every MASS scan you've run so far starts with no memory of previous runs. The Static Analysis agent doesn't remember it flagged the same pattern last week. The Red Team agent doesn't know which techniques were already patched. The Tribunal doesn't know its past verdicts were overturned by the security team. In a production SOC, this is not acceptable — analysts lose time re-litigating decisions that were already made. Agent diaries solve this by giving each agent a persistent, named log that survives between sessions.

AAAK format — compact agent memory. AAAK (Alphanumeric Abbreviated Annotated Key) is MemPalace's index compression dialect. Each diary entry is a pipe-delimited symbolic string. The LLM can scan 1,000 entries in a single context window to find relevant ones — then retrieve only the full drawer it needs. This is how you give an agent months of memory without blowing your token budget on every query.

This is not a toy attack. In production multi-agent systems, if any agent can write to any other agent's memory store, a compromised agent becomes a memory poisoning vector for the entire fleet. Defenses: per-agent write authorization (only the named agent can write to its own diary), entry signing (each entry includes an HMAC tied to the agent's identity), and staleness checks (entries older than N hours trigger re-verification before suppressing investigation). Document which of these MemPalace supports and which you would need to build.
